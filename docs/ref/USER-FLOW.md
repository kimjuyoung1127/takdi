# 탁디장 스튜디오 — 사용자 흐름 (User Flow)

> 버전: 1.0.0
> 최종 수정: 2026-03-05

---

## 전체 흐름 요약

```
STEP 1. 업로드 (사용자 입력)
    ↓
STEP 2. AI 자동 분석 (~10초)
    ↓
STEP 3. 확인 & 수정 + 미리보기
    ↓
STEP 4. 자동 생성 (~75초)
    ↓
STEP 5. 결과 확인 & 다운로드
```

---

## STEP 1. 업로드

사용자가 영상에 필요한 소스를 입력합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  📄 기획 텍스트 (택 1)                                       │
│  ┌───────────────────────────────────────────────────┐      │
│  │  방법 A: TXT 파일 드래그앤드롭 업로드              │      │
│  │  방법 B: 텍스트 영역에 직접 복사+붙여넣기          │      │
│  └───────────────────────────────────────────────────┘      │
│  ※ 제품명, 타깃, 성분, 카피, CTA 등이 포함된 기획안 텍스트  │
│                                                             │
│  🖼️ 이미지 여러 장                                          │
│  ┌───────────────────────────────────────────────────┐      │
│  │  드래그앤드롭 또는 파일 선택                       │      │
│  │  모델컷, 제품컷, 디테일컷, 컬러차트 등             │      │
│  │  순서 자유 (AI가 자동 배치 제안)                    │      │
│  │  지원 형식: JPG, PNG, WebP                         │      │
│  │  최소 1080px 이상 권장                             │      │
│  └───────────────────────────────────────────────────┘      │
│                                                             │
│  🎨 템플릿 선택                                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐          │
│  │  감성   │ │  모던   │ │  강렬   │ │ 미니멀  │          │
│  │ 페이드  │ │슬라이드 │ │ 바운스  │ │  심플   │          │
│  │ 세리프  │ │ 산세리프│ │  볼드   │ │ 라이트  │          │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘          │
│                                                             │
│  🎵 BGM 선택 (택 1)                                         │
│  ┌───────────────────────────────────────────────────┐      │
│  │  방법 A: 내장 BGM 라이브러리에서 선택              │      │
│  │         (미리듣기 + 파형 표시)                     │      │
│  │  방법 B: 사용자 BGM 파일 직접 업로드               │      │
│  │         (MP3/WAV, 업로드 시 자동 비트 분석)        │      │
│  └───────────────────────────────────────────────────┘      │
│                                                             │
│                     [ 🔍 AI 분석 시작 ]                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 입력 항목 정리

| 항목 | 필수 | 입력 방법 | 비고 |
|------|------|-----------|------|
| 기획 텍스트 | O | TXT 업로드 또는 직접 복붙 | 제품 정보, 카피 등 |
| 이미지 | O | 드래그앤드롭 (복수) | 최소 3장, 순서 자유 |
| 템플릿 | O | 4종 중 선택 | 감성/모던/강렬/미니멀 |
| BGM | O | 내장 선택 또는 파일 업로드 | MP3/WAV 지원 |
| 프로젝트명 | △ | 텍스트 입력 | 미입력 시 자동 생성 |

## STEP 2. AI 자동 분석 (~10초)

"AI 분석 시작" 버튼 클릭 후, Ollama 텍스트 LLM이 자동 처리합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  기획 텍스트 읽기                                            │
│    ↓                                                        │
│  Ollama LLM 분석                                            │
│    ├── 제품명 추출                                           │
│    ├── 핵심 카피 분류 (USP, 베네핏, 성분, 이벤트 등)         │
│    ├── CTA(행동 촉구) 추출                                   │
│    └── 슬라이드 자동 구성:                                   │
│        - 이미지 수(N장)에 맞춰 슬라이드 N개 생성              │
│        - 각 슬라이드에 텍스트 자동 배분                       │
│        - 이미지 ↔ 텍스트 최적 매칭 제안                      │
│        - 슬라이드 순서: 후킹 → 본문 → CTA 흐름으로 자동 정렬 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### AI 분석 출력 예시

```
입력: FORENCOS 와일드플라워 벨벳 틴트 기획문서 + 이미지 4장

분석 결과:
  제품명: "FORENCOS 와일드플라워 벨벳 틴트"
  CTA: "지금 바로 만나보세요"

  슬라이드 1 (후킹):
    텍스트: "NEW 와일드플라워 벨벳 틴트"
    이미지: → 이미지1 (메인 배너) 매칭 제안

  슬라이드 2 (소구):
    텍스트: "가장 완벽한 매트 리퀴드 립틴트"
    이미지: → 이미지2 (모델 클로즈업) 매칭 제안

  슬라이드 3 (상세):
    텍스트: "12가지 NEW 컬러 라인업"
    이미지: → 이미지3 (컬러차트) 매칭 제안

  슬라이드 4 (CTA):
    텍스트: "지금 바로 만나보세요"
    이미지: → 이미지4 (제품 클로즈업) 매칭 제안
```

## STEP 4. 자동 생성 (~75초)

"전체 생성" 클릭 후 백그라운드에서 자동 처리됩니다.
SSE(Server-Sent Events)로 실시간 진행률을 표시합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  생성 진행 상황                                              │
│                                                             │
│  ✅ 이미지 크롭 완료 (4장 × 3비율 = 12장)            5초    │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100%          │
│                                                             │
│  🔄 9:16 세로 영상 렌더링 중...                       30초   │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━●━━━━━━━━━━━━━━  72%          │
│                                                             │
│  🔄 1:1 정사각형 영상 렌더링 중...                    30초   │
│  ━━━━━━━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━━━━━━  48%          │
│                                                             │
│  ⏳ 16:9 가로 영상 대기 중...                                │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   0%          │
│                                                             │
│  ✅ 썸네일 생성 완료                                  2초    │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100%          │
│                                                             │
│  🔄 마케팅 스크립트 생성 중...                        10초   │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━●━━━━━━━━━━━━  80%          │
│                                                             │
│  ※ 썸네일 + 스크립트는 영상 렌더와 동시 병렬 처리           │
│  ※ 영상 2개 동시 렌더 (하드웨어에 따라 조절)                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 백그라운드 처리 순서

```
1. 이미지 스마트 크롭 (N장 × 3비율, 병렬)           ~5초
   ↓
2. 동시 실행 ─┬─ 영상 렌더 9:16                      ~30초
              ├─ 영상 렌더 1:1                       ~30초
              ├─ (완료 후) 영상 렌더 16:9             ~30초
              ├─ 썸네일 생성 (sharp 합성)             ~2초
              └─ 마케팅 스크립트 (Ollama LLM)         ~10초
   ↓
3. NAS 자동 저장                                      ~3초
```

### 산출물 상세

| 산출물 | 형식 | 용도 | 스펙 |
|--------|------|------|------|
| 세로 영상 | MP4 (H.264) | 릴스, 쇼츠, 틱톡 | 1080×1920, 30fps, ~15초 |
| 정사각 영상 | MP4 (H.264) | 인스타 피드 | 1080×1080, 30fps, ~15초 |
| 가로 영상 | MP4 (H.264) | 유튜브, 웹 | 1920×1080, 30fps, ~15초 |

| 매니페스트 | JSON | 시스템 메타데이터 | 파일 목록, 크기, 생성 시간 |

## 소요 시간 요약

| 단계 | 소요 시간 | 사용자 행동 |
|------|-----------|------------|
| Step 1: 업로드 | 사용자 속도 | TXT + 이미지 + 템플릿/BGM |
| Step 2: AI 분석 | ~10초 | 대기 |
| Step 3: 확인/수정 | 사용자 속도 | 슬라이드 편집, 미리보기 |
| Step 4: 자동 생성 | ~75초 | 대기 (진행률 확인) |
| Step 5: 결과 확인 | 즉시 | 다운로드, NAS 확인 |

**사용자의 실제 작업 시간:** 업로드 + 확인/수정 = **약 2~3분**
**AI 처리 시간:** 분석 + 생성 = **약 85초**
