# 탁디장 스튜디오 — 주간 요약 가이드

---

## 기록 규칙

- 파일명: `docs/weekly/{YYYY}-W{WW}.md`
- 매주 금요일 또는 해당 주 마지막 작업일에 생성한다
- 주 번호(WW)는 ISO 8601 기준이다
- 해당 주의 일일 기록(`docs/daily/`)을 종합하여 작성한다

---

## 기록 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 주간 목표 달성률 | 필수 | 이번 주 목표 대비 달성 현황 |
| 완료 기능 | 필수 | FEATURE-MATRIX 기준 완료된 기능 ID 목록 |
| 발견된 이슈 | 선택 | 기술적 이슈, 버그, 설계 문제 등 |
| 다음 주 계획 | 필수 | 다음 주 주요 목표와 작업 계획 |

---

## 템플릿

```markdown
# {YYYY}-W{WW} 주간 요약

> 기간: {MM-DD} (월) ~ {MM-DD} (금)

| 목표 | 상태 | 비고 |
|------|------|------|
| {목표 1} | 완료/진행중/미착수 | {비고} |
| {목표 2} | 완료/진행중/미착수 | {비고} |

## 완료 기능

| ID | 기능 | 검증 근거 |
|----|------|-----------|
| {FEAT-ID} | {기능명} | {커밋/테스트 결과} |

(완료 기능이 없으면 "이번 주 완료된 기능 없음" 기재)

## 발견된 이슈

- {이슈 내용과 영향} (없으면 이 섹션 생략)

## 다음 주 계획

- [ ] {계획 1}
- [ ] {계획 2}

## Phase 진행 현황

| Phase | 주초 상태 | 주말 상태 | 변화 |
|-------|-----------|-----------|------|
| {N} | {상태} | {상태} | {진행률 변화} |
```
