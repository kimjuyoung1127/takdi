# korean-ui-i18n

한국어 우선 UI 복구와 앱 내부 i18n 기반 도입 작업 기록입니다.

## 범위
- 사용자 노출 UI를 한국어로 통일
- `src/i18n/` 기반 typed dictionary + provider + hook 추가
- 홈, 공용 레이아웃, 프로젝트/설정 페이지, 컴포즈 공통 다이얼로그와 툴바 문구 치환

## 운영 규칙
- 배치 단위로 구현하고 문서를 갱신
- 사용자 노출 문자열만 바꾸고 기능 동작은 유지
- 신규 파일은 상단 기능 설명 주석 유지
- 마지막 배치에서 `npm run build`까지 확인

## 결과
- 기본 locale은 `ko`
- 외부 i18n 라이브러리 없이 앱 내부 사전 구조 사용
- 향후 `en.ts` 추가가 가능한 구조로 정리
