# 탁디장 스튜디오 — 일일 작업 기록 가이드

---

## 기록 규칙

- 파일명: `docs/daily/{YYYY-MM-DD}.md`
- 작업이 발생한 날마다 해당 날짜의 파일을 생성한다
- 하루에 여러 작업 세션이 있으면 같은 파일에 누적 기록한다
- 기록은 작업 완료 시점에 즉시 작성한다

---

## 기록 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 작업 내용 | 필수 | 수행한 작업 요약 (구체적으로) |
| 변경 파일 | 필수 | 생성/수정/삭제된 파일 경로 목록 |
| 결정 사항 | 선택 | 기술적 결정, 설계 변경 등 |
| 블로커 | 선택 | 진행을 막는 이슈 (없으면 생략) |
| 다음 할 일 | 필수 | 다음 작업 세션에서 할 일 |

---

## 템플릿

```markdown
# {YYYY-MM-DD} 작업 기록

## 작업 내용

- {작업 1 요약}
- {작업 2 요약}

## 변경 파일

- `{파일 경로 1}` — {변경 내용}
- `{파일 경로 2}` — {변경 내용}

## 결정 사항

- {결정 내용과 근거}

## 블로커

- {블로커 내용} (없으면 이 섹션 생략)

## 다음 할 일

- [ ] {할 일 1}
- [ ] {할 일 2}
```
